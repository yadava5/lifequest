# Contributing

Thanks for helping build LifeQuest.

## Branching

- Create a short-lived branch from `main`.
- Use prefixes: `feat/`, `fix/`, `docs/`, `chore/`.
- Keep commits small and focused.

## Pull requests

- Describe what changed and why.
- List testing performed (or explain why not).
- Link relevant docs or screenshots when UI changes are involved.

## Quality checks

- API: `npm run test` (from `apps/api`)
- Desktop: `npm run lint` (from `apps/desktop`)
